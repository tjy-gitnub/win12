# 安全策略

本安全策略适用于 GitHub 上的 tjy-gitnub/win12 仓库。

如需报告与 tjy-gitnub.github.io/win12 提供的服务相关的安全问题，请参阅此文件。

## 安全/漏洞修复版本

安全更新和漏洞修复通常仅在最新版本中提供。

修复程序始终包含在下一个版本中。

安全修复程序享有优先权，并且可能足以促使发布新版本。

## 报告漏洞

我们鼓励负责任地披露安全漏洞。

如果您发现任何可疑之处，我们鼓励并感谢您的报告！

## 可能的安全事项
​​
如果您发现CDN供应链(高度可能)存在投毒，或对任何在本项目使用的API有安全疑虑，并提出足够且合理的理由说明自己的观点，请以以下方式向我们报告：

### 报告方式

为了使漏洞报告能够尽快送达维护人员，首选方式是使用关联 GitHub 项目“Security”选项卡下的“Report a vulnerability”按钮。

这将在报告者和维护人员之间创建一个私密的沟通渠道。

如果您确实无法使用 GitHub 的漏洞报告工作流程，或者有充分的理由不使用，请转到“Issues”选项卡。
